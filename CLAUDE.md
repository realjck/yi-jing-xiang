### Entry point

`index.html` loads globals in this order before `app.js`:
- `assets/libs/marked/marked.min.js` â†’ `marked` (markdown parser)

### Swiper carousel

The UI is a Swiper carousel with up to 5 slides:
1. Welcome screen
2. Coin toss screen
3. Hexagram 1 result
4. Hexagram 2 result (only if mutation)
5. Oracle reader (`#end` slide)

Slides 3â€“5 start `display:none` and are shown after a completed casting.

### Yi Jing casting logic

Coin tosses accumulate via `AddBar()`. After 6 tosses, the 6-character state string (`yiking`, each char: 6/7/8/9) is converted to two 6-bit binary strings:
- **Hexagram 1**: stable lines (7â†’1, 8â†’0, mutatingâ†’stays)
- **Hexagram 2**: changing lines (mutations flipped)

`yiking[i]` maps to position `i+1` (bottom=1, top=6). A mutation occurs when the value is `6` (yin-mut) or `9` (yang-mut).

### Oracle reader

`buildOracleHTML()` (async) fetches and assembles the oracle text:
1. Loads `assets/data/book/{lang}/{nn}.md` for hexagram 1 (number extracted from `HEXAGRAMS_TEXTS`)
2. Renders all `h1` and `h2` sections via `marked.parse()`
3. If mutation: renders only the `h3` sections at mutated positions (h3 blocks are ordered bottomâ†’top, matching `yiking` index 0â†’5); also includes the 7th `h3` ("Tous les traits") if all 6 lines are mutant
4. If mutation: loads hexagram 2's markdown and renders its `h1`/`h2` sections, preceded by an `<hr>`

`parseSections(mdText)` splits raw markdown into `{level, h3Index, markdown}` objects by heading regex `^(#{1,3}) `.

The oracle slide (`#end`) overrides the global `.swiper-slide` flexbox with `display: block` so `#oracle-content` can fill the full height and scroll independently.

### Localization

Two languages (en/fr), persisted in `localStorage` as `YiJingXiang_lang`. Switching language re-renders UI text and reloads the oracle content in the new language if a casting is active.

### PWA / Service Worker

`sw.js` is generated by Workbox CLI (`npm run pwa`). It precaches all static assets (including all `.md` files and `marked.min.js`) for offline-first operation. Regenerate it whenever assets are added or renamed.
