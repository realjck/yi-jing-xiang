# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm install          # Install workbox-cli (only dev dependency)
npm run pwa          # Regenerate sw.js (Service Worker) after asset changes
```

No build step — the app is served directly as static files. Open `index.html` in a browser or use any local HTTP server.

## Architecture

**Pure Vanilla JS PWA** — no bundler, no framework. Production assets are jQuery and Swiper.js (both vendored in `assets/libs/`).

### Entry point

`index.html` loads two global data objects before `app.js`:
- `assets/data/hexagrams-texts.js` → `HEXAGRAMS_TEXTS[lang][binaryString]`
- `assets/data/ui-texts.js` → `UI_TEXTS[lang][key]`

All application logic lives in `assets/app/app.js` (~261 lines). It reads from these globals and from `localStorage` for language and sound preferences.

### Swiper carousel

The UI is a Swiper carousel with three slides (coin toss screen → result screen → book screen). Navigation is handled entirely through Swiper's `slideTo()` API.

### Yi Jing casting logic

Coin tosses accumulate via `AddBar()`. After 6 tosses, the 6-character state string (each char: 6/7/8/9) is converted to two 6-bit binary strings:
- **Hexagram 1**: stable lines (7→1, 8→0, mutating→stays)
- **Hexagram 2**: changing lines (mutations flipped)

These binary strings index into `HEXAGRAMS_TEXTS` and also determine which trigram images to display (filenames are 3-bit binary: `000.jpg` … `111.jpg`).

### Localization

Two languages (en/fr), persisted in `localStorage` as `YiJingXiang_lang`. Switching language re-renders all UI text from `UI_TEXTS` and updates any displayed hexagram title from `HEXAGRAMS_TEXTS`.

### Data

- **Hexagram titles**: `assets/data/hexagrams-texts.js` — 64 entries per language, keyed by 6-bit binary string
- **Book content**: `assets/data/book/en/*.md` and `assets/data/book/fr/*.md` — 64 Markdown files (01.md … 64.md), one per hexagram

### PWA / Service Worker

`sw.js` is generated by Workbox CLI (`npm run pwa`). It precaches all static assets for offline-first operation. Regenerate it whenever assets are added or renamed.
